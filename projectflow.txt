1. Create a git repo and clone it in local setup.
2. Create a mycode.py file and code to it. (it will save a csv file to a new "data" folder)
3. py -3.11 -V
4. py -3.11 -m venv .venv
5. source .venv/Scritps/activate
6. python -m pip install --upgrade pip
7. create a requirements.txt file for libraries
8. pip install -r requirements.txt
9. python mycode.py
10. Do a git add -commit -push before initializing dvc.
11. add dvc to requirements.txt file
12. Now we do "dvc init" (creates .dvcignore, .dvc)
13. Now do "mkdir S3' (creates a new S3 directory)
14. Now we do "dvc remote add -d myremote S3"
15. Next "dvc add data/"

16.  To stop tracking from Git:
            git rm -r --cached 'data'
            git commit -m "stop tracking data"

17. run again "dvc add data/"

18. To track the changes with git, run:
        git add .gitignore data.dvc

19. git status
20. dvc status

21. git log --oneline
22. git add.
23. git commit -m "dvc initiated with data version 1"
24. git push origin main
25. git status

26. dvc is tracking our data but no folder or files in S3 remote:
        dvc commit
        dvc push

27. git add ./ git commit -m 'first version of data saved'/ git push origin main

28. make changes to mycode.py file to append new row in the data. - check changes via dvc status:
        python mycode.py
        dvc status

29. Again -- "dvc commit" and then "dvc push"

30. git add . / git commit -m "second version of data saved" / git push origin main

31. make changes to mycode.py file to append 2nd new row in the data:
            python mycode.py
            dvc status
            dvc commit 
            dvc push

32. git add . / git commit -m "second version of data saved" / git push origin main
